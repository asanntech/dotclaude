# Clean Branches Skill

## 処理フロー

1. **保護ブランチの定義** — 以下のブランチは削除対象から除外する:
   - 現在チェックアウト中のブランチ
   - `main`
   - `release`
   - `staging`
   - `develop`

2. **ブランチ一覧の取得** — `git branch` でローカルブランチの一覧を取得

3. **削除対象の特定** — 保護ブランチを除いた削除対象ブランチをリストアップ

4. **削除対象がない場合** — 「削除対象のブランチはありません」と伝えて終了

5. **ユーザーに確認** — 削除対象のブランチ一覧を表示し、承認を求める

6. **削除の実行** — ユーザーが承認したら `git branch -D` で削除を実行

7. **結果の報告** — 削除結果を報告する

## ユーザーへの確認フォーマット

```
以下のブランチを削除します:

  - feature/xxx
  - fix/yyy
  - ...

保護対象（削除しない）:
  - main
  - develop
  - (現在のブランチ名)

よろしいですか？
```

## 注意事項

- 必ずユーザーの承認を得てから削除を実行する
- 必ず `git branch -D`（強制削除）を使用する。`-d` は絶対に使わないこと
- 削除に失敗したブランチがあれば個別にエラーを報告する
